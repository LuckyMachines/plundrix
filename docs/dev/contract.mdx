## Enums

### `GameMode`

```solidity
enum GameMode { FREE, STAKES }
```

| Value | Int | Description |
|-------|-----|-------------|
| `FREE` | 0 | No entry fee, no prizes. Default for all pre-upgrade games. |
| `STAKES` | 1 | Entry fee required. Prize pool distributed to winner on completion. |

### `Action`

```solidity
enum Action { NONE, PICK, SEARCH, SABOTAGE }
```

| Value | Int | Description |
|-------|-----|-------------|
| `NONE` | 0 | No action (internal use only) |
| `PICK` | 1 | Attempt to crack a lock |
| `SEARCH` | 2 | Search for a tool |
| `SABOTAGE` | 3 | Stun another player and steal a tool |

### `GameState`

```solidity
enum GameState { OPEN, ACTIVE, COMPLETE }
```

### `OutcomeReason`

```solidity
enum OutcomeReason {
    NONE,
    PICK_SUCCESS,          // Lock cracked
    PICK_FAILED_STUNNED,   // Stunned — auto-fail
    PICK_FAILED_ROLL,      // RNG miss
    SEARCH_SUCCESS,        // Tool found
    SEARCH_FAILED_ROLL,    // RNG miss
    SEARCH_FAILED_MAX_TOOLS, // Already at 5 tools
    SABOTAGE_FAILED_INVALID_TARGET,
    SABOTAGE_SUCCESS_STEAL,     // Stunned + stole tool
    SABOTAGE_SUCCESS_STUN_ONLY, // Stunned, attacker at max tools
    SABOTAGE_SUCCESS_NO_TOOL,   // Stunned, target had no tools
    NO_SUBMISSION               // Player didn't submit (non-timeout only)
}
```

## Access Control Roles

```solidity
bytes32 GAME_MASTER_ROLE    // Configure automation, fees
bytes32 AUTO_RESOLVER_ROLE  // Batch-resolve timed-out games
bytes32 RANDOMIZER_ROLE     // Provide external entropy
bytes32 PAUSER_ROLE         // Pause/unpause contract
bytes32 UPGRADER_ROLE       // Upgrade implementation
```

All roles are managed via OpenZeppelin `AccessControlEnumerable`. The `DEFAULT_ADMIN_ROLE` can grant/revoke any role.

## Write Functions

### Game Creation

#### `createGame() → uint256 gameID`

Create a FREE game. No entry fee, no prize pool.

```solidity
function createGame() external whenNotPaused returns (uint256 gameID);
```

#### `createGame(GameMode, uint256) → uint256 gameID`

Create a game with a specific mode. For STAKES, the fee system must be enabled.

```solidity
function createGame(
    GameMode mode,
    uint256 entryFee
) external whenNotPaused returns (uint256 gameID);
```

**Requirements (STAKES):**
- `entryFee > 0`
- `_feeEnabled == true` (call `configureFee` first)
- `_feeRecipient != address(0)`

**Requirements (FREE):**
- `entryFee == 0`

### Registration

#### `registerPlayer(uint256 gameID)`

Join an open game. For STAKES games, send the exact entry fee as `msg.value`.

```solidity
function registerPlayer(uint256 gameID) external payable;
```

| Mode | `msg.value` | Effect |
|------|------------|--------|
| FREE | Must be `0` | Reverts if ETH sent |
| STAKES | Must equal `entryFee` | Added to prize pot |

### Gameplay

#### `startGame(uint256 gameID)`

Start a game. Callable by any registered player or `GAME_MASTER_ROLE`. Requires `>= 2` players.

#### `submitAction(uint256 gameID, Action action, address sabotageTarget)`

Submit your action for the current round. One action per player per round.

- `PICK` and `SEARCH`: pass `address(0)` as `sabotageTarget`
- `SABOTAGE`: must target another registered player (not yourself)

#### `resolveRound(uint256 gameID)`

Resolve the current round. Callable by **anyone** when:
- All players have submitted, OR
- `ROUND_TIMEOUT` (5 minutes) has elapsed

**Default moves:** When resolving after timeout, any player who didn't submit automatically gets a `PICK` action assigned. This ensures games converge toward completion even with AFK players.

### Prize & Withdrawal

#### `withdraw()`

Pull-pattern withdrawal. Withdraws your full `_withdrawable` balance.

```solidity
function withdraw() external;
```

Reverts with `"Nothing to withdraw"` if balance is zero. Uses low-level `call` for ETH transfer (CEI pattern).

### Admin

#### `configureFee(bool feeEnabled, address feeRecipient)`

Enable/disable the fee system. Required before creating STAKES games. `GAME_MASTER_ROLE` only.

#### `configureAutomation(bool autoResolveEnabled, uint256 autoResolveDelay, bool requireExternalEntropy)`

Configure the autoloop worker behavior. `GAME_MASTER_ROLE` only.

#### `provideRoundEntropy(uint256 gameID, uint256 round, uint256 entropy)`

Inject external randomness for a round. `RANDOMIZER_ROLE` only.

#### `resolveTimedOutGames(uint256[] gameIDs) → uint256 resolvedCount`

Batch-resolve timed-out games. `AUTO_RESOLVER_ROLE` only.

## Storage Layout (Upgrade Safety)

The contract uses a `uint256[44] __gap` at the end for future storage slots. When adding new state variables, decrement `__gap` accordingly.

Current storage after inherited slots:

| Slot(s) | Variable | Type |
|---------|----------|------|
| — | `_gameIdCounter` | `Counters.Counter` |
| — | `_games` | `mapping(uint256 => GameInfo)` |
| — | `_players` | `mapping(uint256 => mapping(uint256 => PlayerState))` |
| — | `_playerIndex` | `mapping(uint256 => mapping(address => uint256))` |
| — | `_pendingActions` | `mapping(uint256 => mapping(address => PendingAction))` |
| — | `_roundEntropy` | `mapping(uint256 => mapping(uint256 => uint256))` |
| — | `_autoResolveEnabled` | `bool` |
| — | `_autoResolveDelay` | `uint256` |
| — | `_requireExternalEntropy` | `bool` |
| — | `_feeEnabled` | `bool` |
| — | `_feeRecipient` | `address` |
| — | `_gameMode` | `mapping(uint256 => GameMode)` |
| — | `_entryFee` | `mapping(uint256 => uint256)` |
| — | `_pot` | `mapping(uint256 => uint256)` |
| — | `_withdrawable` | `mapping(address => uint256)` |
| 44 slots | `__gap` | `uint256[44]` |

`GameMode.FREE == 0` means existing games created before the upgrade auto-default to FREE with no migration needed.
